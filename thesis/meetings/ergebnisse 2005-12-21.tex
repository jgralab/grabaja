% D. HAUPTTEIL

\begin{itemize}
  \item{Der Faktenextraktor als PlugIn für Eclipse sollte der letzte Ausweg sein. Besser ist ein Kommandozeilenwerkzeug, welches autark arbeitet.}
  \item{Deshalb mehr Recherche im Bereich der Parser nötig. Wichtig ist ein Werkzeug was einer "`verlässlichen"' Quelle entstammt. D. h., dass es weiterhin gepflegt wird (im Hinblick auf neue Java Versionen - damit wir das in Zukunft nicht selbst machen müssen) und, dass es auch wirklich hält, was es verspricht. Tips:}
  \begin{itemize}
    \item{fujaba}
    \item{Suns Java Compiler oder den gcc "`anzapfen"'}
    \item{DMM, Dargstuhl, Lethbridge}
    \item{AST-Viewer (siehe Hinterwäller-Dipl.-Arbeit)}
    \item{JRefactory}
    \item{JavaCC}
  \end{itemize}
  \item{Der Parser muss nicht unbedingt die aktuellste Java-Version unterstützen, es sollte aber nach Möglichkeit absehbar sein, dass dies in der Zukunft implementiert werden wird}
  \item{Wichtig ist genau zu wissen, was der Parser zurückgibt, um den nächsten Schritt im Graphenaufbau zu kennen.}
  \item{Per Reflection an die Signaturen der class-Dateien herausziehen. Muss nicht unbedingt "`per Hand"' geschehen, vielleicht existieren dafür schon Werkzeuge.}
  \item{Es reicht nur syntaktisch korrekte Programme verarbeiten zu können, eine gewisse Fehlertoleranz der Parser wäre aber wünschenswert, da kein "`reales"' System ohne Fehler ist.}
  \item{Es reicht, wenn nur vorhandene (eigene) Java-Sourcen feingranular auf einen Graphen abgebildet werden. Ausgeschlossen davon sind externe (nicht eigene) Klassen, wie das JDK.}
  \item{Die Besonderheiten von Java beachten, da diese eventuell beim Parsen und der späteren Anfrageverarbeitung zum Tragen kommen (könnten). Beispiele:}
  \begin{itemize}
    \item{nested/inner classes (als Datei mit \$ davor)}
    \item{modifier protected (kann bei "`komischer"' Vererbung zu ungewollten Sichtbarkeiten führen)}
    \item{anonymous classes}
  \end{itemize}
  \item{Die Systemgröße bedingt das Schema. Je größer das System, desto grober sollte das angewandte Schema für den Graphen sein, da sonst die Verarbeitungszeit zu hoch sein könnte.}
  \item{Mehr Dokumentation der Arbeiten. Auch Ergebnisse der Internetsuche, im Hinblick auf schriftlichen Teil der Studienarbeit (Bsp. warum taugt Werkzeug XYZ nicht).}
  \item{Überarbeitung der Anforderungsliste nötig.}
  \item{Ergebnisse der Besprechungen zukünftig dokumentieren.}
  \item{Mehr Kommunikation zu Betreuern nötig, offene Fragen ruhig direkt per E-Mail stellen.}

\end{itemize}
