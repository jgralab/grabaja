\begin{itemize}
	\item{Es sollen folgende Änderungen am Metamodell durchgeführt werden:}
	\begin{itemize}
		\item{Die beiden Elemente ArrayInitializer und ArrayCreation sollen von einander getrennt werden.}
		\item{Es sollen TypeParameterDeclaration-Elemente in Konstruktordefinitionen möglich sein.}
		\item{Falls sich dies nach einer Überprüfung als möglich erweist, sollen Deklarationen im ForIterator-Element ermöglicht werden.}
		\item{Es soll eine InstanceOf-Expression eingeführt werden.}
		\item{RAW-Typen sollen mittels eines Flags identifiziert werden.}
		\item{Die Definitionen von Class, Interface und Enum sollen soweit wie möglich unter der Definition zusammengefasst werden.}
		\item{Die VariableLengthParameterDeclaration soll von der normalen ParameterDeclaration abgeleitet werden.}
		\item{Alle formalen Parameter sollen "`Parameter"' im Namen tragen.}
		\item{Alle aktuellen Parameter sollen "`Argument"' im Namen tragen.}
		\item{Der Bezeichner für das Element des Methodenaufrufs ("`MethodCall"' oder "`MethodInvocation"') soll der Java Language Specification entsprechend benannt werden.}
		\item{Generics-Elemente sollen dadurch identifiziert werden, dass sie eine Kante "`isTypeSpecOf"' auf das entsprechende "`Ident"'-Element besitzen.}
		\item{Das "`Ident"'-Element soll zu "`Indetifier"' umbenannt werden.}
		\item{Alle Methoden sollen als "`MethodDefinition"' bezeichnet werden, nur wenn diese keinen Rumpf besitzen, soll "`MethodDeclaration"' gewählt werden.}
		\item{Beim Methodenaufruf soll nicht zwischen normalen Aufrufen und Aufrufen von Methoden in der Superklasse unterschieden werden.}
		\item{Die Struktur zur Repräsentation der Arraygrenzen ("`Bounds"') soll vereinfacht werden.}
	\end{itemize}
  \item{Das nächste Treffen findet nach Vereinbarung statt.}
  \end{itemize}
